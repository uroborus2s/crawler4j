# 自治规则

- 更新时间：2026-03-27
- 适用范围：`crawler4j` 当前工厂化实施阶段

## 默认执行边界

1. 先读压缩记忆，再按需读正式文档；默认不散读整仓长文。
2. 只处理当前用户要求直接相关的范围，不顺手扩展到无关模块。
3. 发现脏工作区时，不回滚、不清理、不覆盖未知改动；仅在阻塞当前任务时升级沟通。

## 同步规则

1. 任何已接受变更都要同步：
   - 代码
   - `docs/`
   - 测试或验证记录
   - `.factory/memory/`
2. 文档类变更至少同步：
   - 对应目录 `index.md`
   - `docs/traceability/document-index.md`
   - 相关 `.factory/memory/` 状态或映射文件

## 校验规则

1. 代码变更后，优先跑最小必要验证，再决定是否扩大回归范围。
2. 文档变更后，至少检查目录入口完整性和 Markdown 链接有效性。
3. 若命令失败的原因看起来是沙箱、缓存权限、网络或真实环境限制，应直接改用提权或记录为环境阻塞，不误判为代码问题。

## 决策规则

1. 先修事实漂移，再做功能增强。
2. 先修当前控制面和索引，再修外围说明。
3. 优先关闭“声明与实际不一致”的问题，例如入口、版本、导航、发布口径、加载链路。

## 停止条件

- 需要破坏性操作但用户未明确授权。
- 需要写出当前工作区之外且未授权的目录。
- 需要真实站点凭据、浏览器依赖或外部环境条件，而本地事实不足以继续推进。
