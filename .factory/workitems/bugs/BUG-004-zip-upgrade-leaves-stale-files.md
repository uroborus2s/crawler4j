# BUG-004 模块 ZIP 升级会残留旧文件，未满足原子升级/回滚要求

- 状态：DONE
- 类型：BUG
- 优先级：P1
- 估算：1.0 人/天
- 关联 ID：`BUG-004`, `REQ-002`, `TASK-008`
- 发现日期：2026-03-26

## 问题

`ModuleRegistry._install_from_zip()` 曾直接解压到安装目录，不会像目录安装那样先清空目标目录，也没有回滚机制。

## 证据

- 现已改为：先解压到临时目录，校验后备份旧目录，再原子移动到目标目录
- 回归测试 `tests/unit/test_core/test_mms/test_external_module_install.py` 已覆盖升级后旧文件不残留

## 影响

- 已消除“升级后残留旧文件”的主要风险
- 失败时会保留旧版本可用状态

## 验收标准

- ZIP 升级前先进入临时目录校验并替换目标目录
- 同名模块升级后不残留旧版本文件
- 失败时保留旧版本可用状态
- 增加针对 ZIP 升级覆盖/回滚的回归验证
