---
description: 项目文档完善 (Documentation Polish)
---

**触发条件**: 用户要求"完善文档"、"准备发布"或完成大功能后。

**阶段一：文档一致性检查 (角色: @技术文档专家)**
1. **差异对比**: 对比 `src/` 下的代码变更和 `docs/` 下的说明文档。
2. **识别缺失**: 找出缺少参数说明的配置、过时的架构图、未记录的 CLI 命令。

**阶段二：文档更新**
1. **更新对外文档**: 重点更新 `README.md`、`docs/user-guide/` (用户手册) 和 `docs/plugin-dev/` (插件开发文档)。
2. **更新 API 文档**: 根据 Docstring 自动刷新 SDK 文档。
3. **版本记录**: 更新 `CHANGELOG.md`。

**阶段三：润色**
1. 确保语言通顺、排版美观（Markdown 规范）。
2. 检查所有示例代码（Code Snippets）是否能运行。
