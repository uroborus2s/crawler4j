---
description: 项目审查与优化 (Review & Optimize)
---

**触发条件**: 用户要求"审查代码"或"提供优化建议"。

**阶段一：深度审查 (角色: @首席架构师)**
1. **全库扫描**: 检查代码风格、复杂度、硬编码、安全性问题。
2. **架构评估**: 检查是否有模块耦合过紧的情况（尤其是 UI 与 Core 之间）。

**阶段二：提出建议**
1. 生成一份 **[优化建议报告]**，包含：
   - 🔴 高优先级：Bug 风险、安全漏洞。
   - 🟡 中优先级：性能瓶颈、逻辑冗余。
   - 🟢 低优先级：命名规范、注释补充。

**阶段三：实施优化 (需用户确认)**
1. 根据用户选定的优化点进行重构。
2. 重构后必须运行 `pytest` 验证。
