# 03 项目结构与契约

到这一章为止，你已经知道怎么“创建一个模块项目”。  
接下来你要理解的是：宿主应用到底靠什么识别并执行你的模块。

这一步非常重要。  
很多新手第一次做模块时，代码能写、CLI 也会用，但就是反复遇到下面这些问题：

- 明明有 Python 文件，宿主就是找不到模块
- 明明写了工作流，策略里就是选不到
- 明明代码没报错，但调试目标解析失败
- 明明 build 成功，但正式安装不通过

这类问题几乎都和“项目结构与契约”有关。

## 阅读目标

读完本章后，你应该能看懂：

1. 模块目录里每个关键文件是做什么的。
2. 根 `__init__.py` 为什么是运行时入口。
3. `module.yaml` 里哪些字段最关键。
4. 为什么“名字写对”比“代码写多”更重要。

## 推荐阅读顺序

## 这一章最重要的一句话

模块不是“只要有 Python 代码就能跑”，而是“必须满足宿主认可的目录结构和清单契约才能跑”。
