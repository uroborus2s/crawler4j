# 01 概念与约束

这一章是给第一次接触 `crawler4j` 的人准备的“心理建设”章节。

很多人一上来就直接跑 `init-model`，然后边做边猜系统怎么工作。这样当然也能做，但通常会在下面这些地方反复踩坑：

- 不知道模块和宿主应用是什么关系
- 不知道为什么本地能跑、宿主里却跑不起来
- 不知道为什么写了代码也看不到 `🐞 调试`
- 不知道为什么能 build 包，却不能正式安装

本章就是先把这些基础认知补上。

## 阅读目标

读完本章后，你应该能回答下面这些问题：

1. `crawler4j` 模块到底是什么。
2. 模块项目和宿主应用的边界是什么。
3. 当前版本有哪些不能绕过的真实限制。
4. 第一次开发模块时，哪些方向应该先做，哪些方向先不要碰。

## 推荐阅读顺序

1. [1.1 系统地图与术语](./01-system-map.md)
2. [1.2 当前真实约束](./02-real-constraints.md)

## 如果你很着急

即使你想直接开始敲代码，也建议至少先读完这一章，再进入 [02 快速开始](../02-quickstart/index.md)。  
因为后面很多“看起来像 bug”的现象，其实都只是这里提前说明过的系统约束。
