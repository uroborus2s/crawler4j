# 04 模块开发

这一章才是真正开始“写模块代码”。

但即使到了这里，也不建议你一上来就把整个业务流程全部写完。  
对第一次开发模块的人，最稳的做法永远是：

1. 先让一个最小任务脚本跑起来
2. 再让一个最小工作流调到这个任务
3. 最后再补配置、命令和 UI 细节

## 阅读目标

读完本章后，你应该能：

1. 新建一个任务脚本并看懂它的执行接口。
2. 新建一个工作流并通过 `ctx.run_subtask()` 调子任务。
3. 用 CLI 补齐工作流和 UI 相关文件。
4. 知道第一次开发模块时，哪些能力先不要过度设计。
