# 05 调试

这一章只讲当前真实可用的调试链路，不讲历史上的离线调试脚本，也不讲已经被替换掉的临时方案。

第一次开发模块的人，最容易在调试上产生两个误解：

1. “我本地有源码，所以宿主应该能调试我。”
2. “只要 IDE 配好了，`🐞 调试` 就会出现。”

这两件事都不对。  
当前调试链路的关键前提是 `DevLink` 和策略解析，而不是 IDE 本身。

## 阅读目标

读完本章后，你应该知道：

1. `DevLink` 到底是什么。
2. 为什么只有 `DevLink` 模块才会出现真实调试入口。
3. 策略中的哪些字段直接决定调试能不能发起。
4. 看不到 `🐞 调试` 时应该先查什么。
