# 02 快速开始

这一章的目标不是“把所有概念讲完”，而是“先让你亲手把一条最小链路跑通”。

如果你是第一次接触这个项目，请把这章理解成：

- 一套最少命令
- 一条最短路径
- 一次低风险试跑

你先完成一次最小成功体验，后面读结构和契约时会轻松很多。

## 阅读目标

完成本章后，你应该至少拥有：

- 一个标准模块项目目录
- 一个额外任务脚本
- 一个额外工作流
- 一个基础配置 UI 文件
- 一条可以接进 Core 的最小调试链路

## 推荐阅读顺序

1. [2.1 开发前准备](./01-environment-setup.md)
2. [2.2 创建第一个模块](./02-create-first-module.md)

## 这章做完不代表什么

请注意，这一章完成后，只代表你“已经把第一条链路跑通”。  
它不代表你已经理解：

- 模块为何能被识别
- 工作流为何能被选中
- 为什么调试依赖 `DevLink`
- 为什么交付必须再做一次 zip 安装验收

这些问题会在后面的章节中补齐。
