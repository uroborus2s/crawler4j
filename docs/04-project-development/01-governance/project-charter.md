## 1. 项目背景

- 蛛行演略（crawler4j）是一个基于 Python 的桌面自动化任务运行平台，采用 Core + Modules + SDK + Contracts 的组合形态。
- 仓库已经存在较完整的代码、测试、文档和历史发布产物，但此前并未按软件工厂规范维护。
- 当前目标不是重写系统，而是先把项目接入统一的工厂治理方式，建立后续迭代的真实基线。

## 2. 目标与成功标准

- 目标 1：基于当前仓库的真实证据建立可交接、可追踪的软件工厂基线。
- 目标 2：明确当前真正可运行、可构建、可发布的链路，以及当前阻塞发布的差异项。
- 目标 3：把下一批实施工作收敛为可执行的 `BUG` / `CR` / `TASK`。

| 指标 | 目标值 | 度量方式 |
|---|---|---|
| 基线文档可读 | 达成 | 已收敛为 `docs/01-getting-started/`、`docs/02-user-guide/`、`docs/03-developer-guide/`、`docs/04-project-development/` |
| 工厂控制面可用 | 达成 | 新增 `.factory/project.json`、memory、process、workitems |
| 真实状态可验证 | 达成 | 至少记录测试、文档构建、打包、入口验证结果 |

## 3. 范围与非目标

### 范围内

- 核心桌面应用、运行时服务、外部模块开发链路、SDK、Contracts、Markdown 文档树、软件工厂治理基线
- 当前仓库已有实现与已有发布产物的事实核查
- 首批缺陷、变更请求、实施任务的创建

### 范围外

- 在本次基线阶段新增业务功能
- 完整重写历史设计文档
- 修复所有 lint/结构债务
