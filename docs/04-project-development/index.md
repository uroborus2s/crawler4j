# 项目开发文档（内）概览

本目录收纳内部治理、需求、设计、实施、测试、发布、运维和追踪矩阵等正式开发文档。它是生产事实源，不是补充说明区。

## 子目录职责

| 目录 | 作用 |
|---|---|
| `01-governance/` | 项目章程、范围、风险和治理边界 |
| `02-discovery/` | 输入证据、现状分析和方案探索 |
| `03-requirements/` | PRD、需求分析和需求校验 |
| `04-design/` | 技术选型、架构、模块边界和接口契约 |
| `05-development-process/` | 软件开发流程、实施方案、执行记录 |
| `06-testing-verification/` | 测试计划、质量门和设计实现审查 |
| `07-release-delivery/` | 验收、发布说明、交付包、版本治理 |
| `08-operations-maintenance/` | 部署说明、运行手册、维护者指南 |
| `09-evolution/` | 流程改进、模式修复和演进计划 |
| `10-traceability/` | 需求矩阵、接口矩阵和文档索引 |

## 使用原则

- 先看所在阶段目录，再看单页，不跨阶段混读。
- 目录树和页面权限由根 `docs/index.md` 声明，这里只负责解释“每个子目录为什么存在”。
