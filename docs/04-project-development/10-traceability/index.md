# 追踪矩阵概览

本目录用于快速回答三个问题：需求落到哪里、接口由谁负责、页面入口是否完整。

## 关键页面

| 页面 | 作用 |
|---|---|
| `requirements-matrix.md` | 需求到设计、任务、测试和发布状态的映射 |
| `interface-matrix.md` | 接口到提供方、消费方、版本、测试和运维责任的映射 |
| `document-index.md` | 正式文档入口和按角色快速阅读路径 |

## 建议阅读顺序

1. 先看 [文档索引](document-index.md)，定位正式入口。
2. 需要核对需求覆盖时看 [需求追踪矩阵](requirements-matrix.md)。
3. 需要核对接口责任时看 [接口追踪矩阵](interface-matrix.md)。
