## 1. 目的

本文件用于收敛“模式级问题”的后续动作，避免同一类文档、导航或流程缺陷在多个项目里反复出现。

## 2. 什么问题应该进入本计划

1. 根导航、索引或 memory 映射经常不同步。
2. 正式文档缺少最小元数据，导致接手人无法快速判断能否使用。
3. 目录中长期存在空壳页，形成伪完成状态。
4. 同类检查需要靠人工重复发现，而不是通过固定校验脚本暴露。

## 3. 演进工作流

1. 先在执行记录或检查结果中留下真实证据。
2. 判断这是单次问题还是模式级问题。
3. 如果是模式级问题，优先修模板、脚本、导航规则或 memory 同步规则。
4. 回归验证通过后，再把结论沉淀到 `.factory/memory/evolution-baseline.md`。

## 4. 当前优先演进项

| 项目 | 当前问题 | 目标 |
|---|---|---|
| 根导航覆盖检查 | `docs/index.md` 可能漏挂正式页面 | 固化“根导航覆盖所有正式 Markdown 页面”的校验 |

## 5. 完成判定

- 问题不再依赖聊天记录提醒，而是由正式规则或校验显式约束。
- 新接手者可以只凭正式文档和 memory 继续推进，而不需要回看历史会话。
