# 0.4.0 使用者/开发者指南版本分流需求方案

**状态：** Draft  
**版本目标：** crawler4j 0.4.0  
**最后更新：** 2026-04-30  
**上游输入：** 0.3.0 稳定文档、0.4.0 `core-native-v2` 方案、docs-stratego 源仓接入、开发者指南、使用者指南  
**下游输出：** `docs/04-project-development/04-design/0.4.0-guide-versioning-architecture.md`

## 1. 背景

当前 `docs/02-user-guide/` 和 `docs/03-developer-guide/` 是单套文档，0.3.x 稳定能力与 0.4.0 开发中能力会在同一目录下持续混写。随着 0.4.0 引入 `core-native-v2`、装饰器对象装配、workflow 无参数、page action 纯函数等破坏性开发者契约，单套指南会让使用者和模块开发者无法判断自己该按哪个版本操作。

0.4.0 需要把使用者指南和开发者指南按版本彻底拆开：0.3.0 文档冻结当前发布契约，0.4.0 文档承载开发中契约和迁移说明。docs-stratego 网站上的主文档必须始终指向当前已发布版本；老版本文档继续保留在网站上可访问；未发布的 0.4.0 只能作为开发版 / 预览版入口出现。内部项目开发文档仍按当前 `04-project-development/` 维护，不进入本轮版本分流范围。

## 2. 目标

| ID | 目标 |
|---|---|
| `G-0401-001` | `02-user-guide` 按 `v0.3.0` / `v0.4.0` 拆分，根目录只保留版本选择入口 |
| `G-0401-002` | `03-developer-guide` 按 `v0.3.0` / `v0.4.0` 拆分，根目录只保留版本选择入口 |
| `G-0401-003` | 0.3.0 指南不得混入 0.4.0 开发中契约，0.4.0 指南不得覆盖 0.3.0 稳定指南 |
| `G-0401-004` | 根导航能清楚区分稳定版指南和开发版指南 |
| `G-0401-005` | 文档校验能发现跨版本链接、未归档页面和版本元数据缺失 |
| `G-0401-006` | docs-stratego 网站主入口默认展示当前已发布版本，历史版本继续可访问 |

## 3. 范围

### 3.1 包含

- `docs/02-user-guide/` 使用者指南版本分流。
- `docs/03-developer-guide/` 开发者指南版本分流。
- 根 `docs/index.md` 导航改为版本入口。
- 每个指南版本新增版本元数据。
- 文档校验增加版本隔离检查。
- docs-stratego 源仓导航规则新增“当前发布版本 / 开发版 / 历史版本”三类入口。

### 3.2 不包含

- 不要求 `docs/04-project-development/` 内部项目开发文档按版本拆分。
- 不要求 `.factory/memory/` 按完整版本复制；memory 只记录当前协作状态和版本分流映射。
- 不要求 0.3.0 文档继续跟随 0.4.0 需求演进。

## 4. 用户故事

### REQ-0401-001 使用者按版本阅读稳定指南

作为宿主使用者，我希望能明确选择 0.3.0 或 0.4.0 使用者指南，以便不会把开发中版本的 UI、运行模板或配置说明误用于稳定版本。

验收标准：

- AC-1: 给定用户打开 `docs/02-user-guide/index.md`，当查看正文和导航，则只能看到版本选择说明，不出现具体操作步骤。
- AC-2: 给定用户选择 0.3.0 使用者指南，则所有操作说明来自 `docs/02-user-guide/v0.3.0/`。
- AC-3: 给定用户选择 0.4.0 使用者指南，则所有操作说明来自 `docs/02-user-guide/v0.4.0/`。

### REQ-0401-002 模块开发者按版本阅读开发者指南

作为模块开发者，我希望能明确选择 0.3.0 或 0.4.0 开发者指南，以便按对应 runtime 契约编写模块。

验收标准：

- AC-1: 给定开发者打开 `docs/03-developer-guide/index.md`，当查看正文和导航，则只能看到版本选择说明，不出现单一版本的开发步骤。
- AC-2: 给定开发者选择 0.3.0 开发者指南，则文档描述 `core-native-v1`、`module.yaml`、`tasks/*.py`、`workflows/*.py` 等稳定契约。
- AC-3: 给定开发者选择 0.4.0 开发者指南，则文档描述 `core-native-v2`、装饰器对象装配、`@page_action`、`@data_table` 等新契约。

### REQ-0401-003 版本内容物理隔离

作为维护者，我希望 0.3.0 和 0.4.0 指南页面在文件路径上物理隔离，以便修改 0.4.0 文档不会意外改变 0.3.0 稳定说明。

验收标准：

- AC-1: 给定 `docs/02-user-guide/`，除 `index.md` 外，不允许直接存在未归入版本目录的 Markdown 页面。
- AC-2: 给定 `docs/03-developer-guide/`，除 `index.md` 外，不允许直接存在未归入版本目录的 Markdown 页面。
- AC-3: 给定 0.3.0 指南页面，当扫描内部链接，则默认只能链接到同一版本目录或公共根入口。
- AC-4: 给定 0.4.0 指南页面，当扫描内部链接，则默认只能链接到同一版本目录、公共根入口或显式标记的迁移对比页面。

### REQ-0401-004 文档版本元数据

作为文档维护者，我希望每个指南版本都有机器可读的版本元数据，以便导航、校验和发布站点能稳定识别文档版本。

验收标准：

- AC-1: 给定 `docs/02-user-guide/v0.3.0/`、`docs/02-user-guide/v0.4.0/`、`docs/03-developer-guide/v0.3.0/`、`docs/03-developer-guide/v0.4.0/`，每个目录都包含版本元数据文件。
- AC-2: 版本元数据至少包含 `doc_version`、`product_version`、`status`、`runtime_api`、`audience`。
- AC-3: 当元数据缺失或版本号与目录名不一致，文档校验失败。

### REQ-0401-005 docs-stratego 主文档指向当前发布版本

作为网站读者，我希望打开 docs-stratego 上的 crawler4j 文档时默认看到当前已发布版本，以便不会误读开发中版本的使用说明或模块开发契约。

验收标准：

- AC-1: 给定当前已发布指南版本为 0.3.0，当 docs-stratego 同步源仓文档，则站点主入口默认链接到 `v0.3.0` 使用者指南和开发者指南。
- AC-2: 给定 0.4.0 尚未正式发布，当 docs-stratego 同步源仓文档，则 0.4.0 只出现在“开发版 / 预览版”入口，不作为主文档。
- AC-3: 给定 0.4.0 正式发布，当维护者执行发布文档提升动作，则 `v0.4.0` 变为主文档，`v0.3.0` 移入历史版本入口并继续可访问。
- AC-4: 给定任一历史版本文档目录存在，当 docs-stratego 校验和构建站点，则历史版本 URL 不被删除或覆盖。

## 5. 非功能需求

| ID | 类型 | 要求 | 验证方式 |
|---|---|---|---|
| `NFR-0401-001` | 可维护性 | 0.3.0 与 0.4.0 指南不得共用 Markdown 正文文件 | 文档校验 |
| `NFR-0401-002` | 可读性 | 根指南入口必须先解释“稳定版 / 开发版”差异，再给版本链接 | 文档审查 |
| `NFR-0401-003` | 可追溯 | 每个版本目录必须有版本元数据，且根导航必须显式显示版本号 | docs-stratego + 自定义校验 |
| `NFR-0401-004` | 迁移安全 | 0.3.0 指南冻结后只允许勘误，不允许新增 0.4.0 新能力说明 | review gate |
| `NFR-0401-005` | 发布一致性 | docs-stratego 主入口版本必须与当前已发布版本事实一致，不得指向未发布版本 | 发布检查清单 |

## 6. 开放问题

| ID | 问题 | 当前建议 |
|---|---|---|
| `OQ-0401-001` | 0.3.0 目录是否命名为 `0.3.0` 还是 `v0.3.0` | 使用 `v0.3.0`，URL 里能明确它是版本目录 |
| `OQ-0401-002` | 是否保留 `latest` 或 `current` 软入口 | 源仓不保留，避免漂移；发布站点可在构建层做跳转 |

| `OQ-0401-004` | docs-stratego 是否需要按 Git tag 拉历史文档 | 不需要。历史版本文档保留在源仓 `docs/02-user-guide/v*/` 和 `docs/03-developer-guide/v*/` 路径中，docs-stratego 继续从主分支同步当前文档树 |
