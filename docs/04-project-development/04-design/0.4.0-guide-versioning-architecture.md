# 0.4.0 使用者/开发者指南版本分流架构方案

**状态：** Draft  
**版本目标：** crawler4j 0.4.0  
**最后更新：** 2026-04-30  
**上游输入：** `0.4.0-guide-versioning-requirements.md`、docs-stratego 源仓接入、当前 `docs/index.md`  
**下游输出：** docs-stratego 导航调整、指南目录迁移任务、发布检查清单更新

## 1. 设计结论

只对使用者指南和开发者指南做版本分流：

- `docs/02-user-guide/` 只保留版本选择入口和各版本使用者指南。
- `docs/03-developer-guide/` 只保留版本选择入口和各版本开发者指南。
- `docs/04-project-development/` 仍是内部过程文档，不按版本复制。
- docs-stratego 继续从源仓 `main` 同步 `docs/index.md`，不按 Git tag 拉历史文档。
- 网站主文档默认指向当前已发布版本；历史版本在站点中保留；开发中版本单独作为预览入口。

核心原则：版本用路径隔离，主入口用导航决定。

## 3. 版本元数据

每个版本目录必须包含 `version.yaml`：

```yaml
doc_version: "0.3.0"
product_version: "0.3.0"
status: "current"
site_role: "main"
runtime_api: "core-native-v1"
audience: "user"
```

字段说明：

| 字段 | 值 | 说明 |
|---|---|---|
| `doc_version` | `0.3.0` / `0.4.0` | 文档版本 |
| `product_version` | 对应产品版本 | 与发布版本事实对齐 |
| `status` | `current` / `development` / `archived` | 当前发布、开发中、历史归档 |
| `site_role` | `main` / `preview` / `archive` | docs-stratego 站点入口角色 |
| `runtime_api` | `core-native-v1` / `core-native-v2` | 开发者指南必须明确 |
| `audience` | `user` / `developer` | 指南类型 |

0.3.0 当前建议：

- 使用者指南：`status=current`、`site_role=main`。
- 开发者指南：`status=current`、`site_role=main`、`runtime_api=core-native-v1`。

0.4.0 未发布前：

- 使用者指南：`status=development`、`site_role=preview`。
- 开发者指南：`status=development`、`site_role=preview`、`runtime_api=core-native-v2`。

## 4. docs-stratego 导航策略

根 `docs/index.md` 仍是 docs-stratego 的唯一导航事实源。导航应按站点读者视角组织：

```yaml
mkdocs:
  nav:
    - title: 当前发布版本
      children:
        - title: 使用者指南 0.3.0
          path: 02-user-guide/v0.3.0/index.md
          access: public
        - title: 开发者指南 0.3.0
          path: 03-developer-guide/v0.3.0/index.md
          access: private
    - title: 开发中版本
      children:
        - title: 使用者指南 0.4.0
          path: 02-user-guide/v0.4.0/index.md
          access: public
        - title: 开发者指南 0.4.0
          path: 03-developer-guide/v0.4.0/index.md
          access: private
    - title: 项目开发文档（内）
      children:
        ...
```

发布后的提升规则：

1. 将 `v0.4.0/version.yaml` 从 `development/preview` 改为 `current/main`。
2. 将 `v0.3.0/version.yaml` 从 `current/main` 改为 `archived/archive`。
3. 根导航把 `0.4.0` 移到“当前发布版本”。
4. 根导航把 `0.3.0` 移到“历史版本”。
5. 不删除 `v0.3.0/` 目录。

不在源仓维护 `latest` 或 `current` 目录，避免 docs-stratego 同步后出现软入口漂移。

## 5. 内容边界

### 5.1 使用者指南

0.3.0 使用者指南应描述当前发布版真实 UI 和操作路径：

- 安装与第一次打开。
- 配置中心和外部浏览器配置。
- 任务、运行模板、环境、资源池等当前稳定能力。
- 当前发布版可见的异常案例和管理员操作。

0.4.0 使用者指南只描述 0.4.0 用户可见变化：

- 运行模板从 workflow 参数配置切到对象装配配置时的 UI 变化。
- 任务运行和资源选择语义变化。
- 旧模板迁移或兼容提示。

### 5.2 开发者指南

0.3.0 开发者指南固定为 `core-native-v1`：

- `module.yaml` 仍是模块静态清单事实源。
- `tasks/*.py -> TASK/execute`。
- `workflows/*.py -> WORKFLOW/run`。
- `pages/*.py` / `pages/<group>/*.py -> PAGE/handler`。
- 数据资源仍按当前 manifest 驱动契约说明。

0.4.0 开发者指南固定为 `core-native-v2`：

- `@interface`、`@component`、`@workflow`、`@page_action`、`@data_table`、`@data_view`。
- workflow 只接收宿主注入对象，不接收 parameters。
- 对象参数只属于 component 创建。
- page action 是页面操作纯函数。
- SDK 在模块打开、DevLink、`check full`、manifest lock 和打包阶段执行装饰器诊断。

## 6. 迁移步骤

1. 使用 `git mv` 将当前 `docs/02-user-guide/*.md` 移到 `docs/02-user-guide/v0.3.0/`。
2. 使用 `git mv` 将当前 `docs/03-developer-guide/*.md` 移到 `docs/03-developer-guide/v0.3.0/`。
3. 为 `02-user-guide/index.md` 和 `03-developer-guide/index.md` 重写版本选择页。
4. 从 `v0.3.0/` 复制出 `v0.4.0/` 初始目录，再只在 0.4.0 目录内改写新契约。
5. 为四个版本目录补齐 `version.yaml`。
6. 更新 `docs/index.md` 的 docs-stratego 导航。
7. 更新 `.factory/memory/doc-map.md`，让版本化页面继续映射到对应 memory 摘要。
8. 运行 `docs-stratego source validate --repo-path .`。

## 7. 校验规则

基础校验：

- `docs-stratego source validate --repo-path .` 必须通过。
- `docs/02-user-guide/` 根目录除 `index.md` 外不能有 Markdown 正文。
- `docs/03-developer-guide/` 根目录除 `index.md` 外不能有 Markdown 正文。
- 每个 `v*/` 目录必须有 `version.yaml`。
- `version.yaml.doc_version` 必须与目录名一致。

链接校验：

- `v0.3.0` 页面默认只能链接到 `v0.3.0` 同类指南、根版本选择页或项目开发文档。
- `v0.4.0` 页面默认只能链接到 `v0.4.0` 同类指南、根版本选择页、项目开发文档或显式迁移对比页面。
- 0.3.0 页面不能链接到 0.4.0 开发者契约页面。

发布校验：

- `site_role=main` 的指南版本只能有一个。
- `site_role=main` 必须与当前正式发布版本一致。
- `status=development` 的版本不得出现在“当前发布版本”导航组。

## 8. 风险

| 风险 | 缓解 |
|---|---|
| 0.4.0 未发布内容误上主文档 | 根导航按 `site_role=main/preview` 分组；发布检查清单校验主入口版本 |
| 0.3.0 勘误时误改 0.4.0 | 版本路径物理隔离；同名页面分别维护 |
| 链接迁移量大 | 先用 `git mv` 保留历史，再批量检查相对链接 |
| docs-stratego 只读根索引导致版本入口漂移 | 根 `docs/index.md` 仍是唯一导航事实源，版本切换只改根导航和 `version.yaml` |
