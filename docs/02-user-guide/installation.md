# 安装说明

当前项目同时支持“从源码运行宿主应用”和“运行已经打好的桌面包”。对于仓库协作者和维护者，当前更推荐先走源码运行；对于只使用应用的人，如果你已经拿到构建产物，可以直接打开打包后的应用。

## 1. 选择安装方式

| 方式 | 适用人群 | 当前建议 |
|---|---|---|
| 从源码运行 | 维护者、协作者、模块开发者 | 推荐，最贴近当前仓库事实 |
| 运行打包产物 | 只使用宿主应用的人 | 可以使用，但问题排查仍要回到源码文档 |

## 2. 从源码运行宿主应用

### 前置条件

- Python `>=3.12`
- `uv`
- 当前仓库源码

### 步骤

```bash
uv sync --all-packages
uv run python -m src.ui.app
```

如果你需要显式使用模块入口，也可以执行：

```bash
uv run python -m src.ui.app
```

### 首次启动会发生什么

应用启动时会自动：

- 初始化 `config.db`、`data.db` 和 `state.db`
- 创建日志目录
- 启动 REM 和 ATM 相关核心服务
- 打开桌面 Shell

## 3. 运行打包产物时要知道什么

- 项目已经验证过 PyInstaller 打包链路可以通过。
- 如果你拿到的是 `.app`、`.exe` 或其他桌面产物，直接启动应用即可，不需要先安装 Python。
- 这类产物仍然会把运行数据写到同一个应用数据目录，而不是写回仓库目录。

## 4. 应用数据会写到哪里

宿主运行时数据位于平台相关的应用数据目录：

- Windows: `%APPDATA%/Crawler4j/`
- macOS: `~/Library/Application Support/Crawler4j/`
- Linux: `~/.local/share/Crawler4j/`

首次启动后，通常会看到下面这些内容：

- `config.db`：系统设置和模块配置
- `data.db`：模块数据表 schema 和 dataset records
- `state.db`：运行态状态和任务相关数据
- `logs/`：运行日志
- `modules/`：正式安装模块目录

## 5. 安装后如何校验

最少确认下面几件事：

1. 应用窗口能正常打开。
2. 应用数据目录已经创建。
3. `config.db`、`data.db`、`state.db` 和 `logs/` 已出现。
4. `📦 模块管理` 页面可以正常打开。

如果你是仓库维护者，建议再补跑一次：

```bash
uv run python scripts/smoke_test_ui.py
```

如果这是一个全新的本地应用数据目录，先至少启动一次宿主应用，再执行这条 smoke 命令。

## 6. 安装时最常见的误判

- 看到仓库里的 `packages/crawler4j/modules/` 目录，不代表模块已经“安装到宿主”。
- 模块项目自己的虚拟环境安装成功，不代表宿主应用已经具备相同依赖。
- 能 `uv build` 出 wheel，不代表宿主正式安装链路已经打通；正式模块交付仍以 zip 安装为准。
