# Crawler4j 用户使用说明书

本文档提供了 Crawler4j 自动化爬虫系统的详细使用指南，涵盖账号管理、防封号策略、操作流程及安全注意事项。

---

## 1. 账号管理详解

系统采用 **携程账号** 与 **劳保账号** 分离的双账号池设计，以最大化任务效率并降低封号风险。

### 1.1 账号体系说明

| 账号类型 | 用途 | 说明 |
| :--- | :--- | :--- |
| **携程账号** (Ctrip) | 用于登录携程旅行网，作为爬虫的入口身份。 | **核心资源**。需绑定手机号，支持接码平台自动获取验证码。 |
| **劳保账号** (Labor) | 用于登录劳保用品平台，执行具体的答题、浏览任务。 | **消耗性资源**。每次运行动态分配，需大量储备。 |

### 1.2 账号导入指南

支持通过 CSV 文件批量导入账号。请在 CSV 文件包含以下列头（标题行）：

#### 携程账号导入格式
- 必填列：`phone` (或 `phone_number`)
- 选填列：
    - `password`: 密码
    - `country_code`: 国家代码 (默认 `+86`)
    - `sms_platform_type`: 接码平台类型
    - `sms_platform_url`: 接码平台API地址
    - `sms_platform_key`: 接码平台API密钥

#### 劳保账号导入格式
- 必填列：`phone`, `password`

### 1.3 账号状态管理

- **空闲 (Idle)**: 账号可用，等待任务分配。
- **已绑定 (Active)**: 账号当前已绑定到某个环境，正在被使用。
- **置黑 (Blacklisted)**: 账号已被检测为异常（如登录失败过多、被反爬检测），系统将不再自动调度该账号，需人工解封或删除。
- **冷却中**: API账号首次登录后会进入 **2天冷却期**，期间不可用于自动任务。

## 2. 防封号策略

系统内置多重防封号机制，模拟真实用户行为。

### 2.1 环境隔离与指纹浏览器
每个运行实例（Environment）都是一个独立的浏览器环境：
- **指纹保护**: 使用 `BitBrowser` 或内置 `VirtualBrowser` 生成独立的设备指纹（UserAgent, Canvas, WebGL等），防止被关联。
- **环境隔离**: 每个环境拥有独立的 Cookies 和 LocalStorage，互不干扰。

### 2.2 智能调度策略
- **并发控制**: 默认最大并发数为 **10**（可在设置中调整，最大 20）。避免短时间产生过大流量。
- **任务间隔**: 任务执行间隔默认为 **5秒**（可调整）。模拟人工阅读停顿，避免机器特征。
- **API账号冷却**: 新注册的 API 账号在首次登录后，系统会自动强制 **冷却 2 天**，以更接近真实用户行为，提高存活率。

### 2.3 最佳实践
- **每日限额**: 建议单日单环境启动次数不超过 20 次。
- **账号轮换**: 保持充足的账号池储备，让系统自动轮换使用，避免单账号高频操作。
- **IP 代理**: 建议配置高质量代理 IP（支持自动轮换），防止 IP 被封禁。

## 3. 操作指南

### 3.1 安装与启动
确保已安装 Python 3.10+ 和 `uv` 包管理器。

```bash
# 1. 安装依赖
uv sync

# 2. 启动 GUI 界面
uv run src/main.py
```

### 3.2 首次运行流程
1. **添加/导入账号**: 进入“携程账号”和“劳保账号”页面，导入您的账号数据。
2. **配置设置**: 在“设置”页面配置浏览器类型（推荐 BitBrowser）、并发数等。
3. **配置接码平台**: 如使用 API 自动接码，请在设置中配置接码平台信息（Host/User/Pass/ProductId）。
4. **启动调度器**: 在“仪表盘”点击 **“启动调度器”** 按钮。系统将自动创建环境并开始执行任务。

### 3.3 环境管理
- **自动模式**: 调度器会自动管理环境的创建、运行和销毁。
- **手动模式**: 
    1. 进入“环境管理”页面。
    2. 点击“+ 新建环境”，手动选择账号。
    3. 点击“启动”按钮，浏览器将打开。此时您可以人工介入操作或观察自动化流程。
    4. **注意**: 手动模式下启动环境会自动锁定所选劳保账号；关闭浏览器后自动释放。

### 3.4 常见问题
- **Q: 浏览器无法启动？**
  - A: 请检查“设置”中的浏览器路径或 API 地址是否正确。如果使用 BitBrowser，确保其客户端已打开。
- **Q: 账号一直显示登录失败？**
  - A: 检查账号密码是否正确，或接码平台余额是否充足。连续失败会导致账号被暂时置黑。
- **Q: 任务执行一半停止？**
  - A: 可能是网络超时或页面结构变更。请查看日志（控制台输出）获取详细错误信息。
